# ❓ Perguntas Frequentes (FAQ)\n\n## 🎯 Geral\n\n### O que é Check-up de TI?\n\nÉ um script PowerShell que automatiza o monitoramento de dispositivos de rede importantes, gerando um relatório visual em HTML com o status de cada um (online/offline).\n\n### Como funciona?\n\nO script:\n1. Testa conectividade com cada dispositivo via PING\n2. Coleta o resultado\n3. Gera um arquivo HTML com tabela de status\n4. Salva automaticamente na sua Desktop\n\n### Preciso de permissões especiais?\n\nNão, mas se a política de execução do PowerShell estiver bloqueada, você pode alterar com:\n```powershell\nSet-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser\n```\n\n## 📦 Instalação\n\n### Como instalar?\n\nTrês formas:\n1. **Mais fácil:** Clique duplo em `executar.bat`\n2. **Com PowerShell:** `.\\Check-upTI.ps1`\n3. **Com Git:** `git clone` + executar\n\nVeja [INSTALL.md](INSTALL.md) para detalhes.\n\n### Qual versão do PowerShell preciso?\n\n**Mínimo:** PowerShell 3.0\n**Recomendado:** PowerShell 5.1+ ou 7.0+\n\nVerifique com: `$PSVersionTable`\n\n### Funciona em Mac ou Linux?\n\nNão. É específico para Windows com PowerShell.\n\nPara Linux, você precisaria adaptar ou usar bash scripts.\n\n## 🔧 Uso\n\n### Como personalizar os dispositivos testados?\n\nEdite `Check-upTI.ps1` e procure pela seção \"CONFIGURAÇÕES\":\n\n```powershell\n$Resultados = Get-StatusHTML -IP \"192.168.1.1\" -Nome \"Roteador\"\n$Resultados += Get-StatusHTML -IP \"seu-ip\" -Nome \"Seu Dispositivo\"\n```\n\n### Posso adicionar mais dispositivos?\n\nSim! Apenas adicione mais linhas com:\n```powershell\n$Resultados += Get-StatusHTML -IP \"xxx.xxx.xxx.xxx\" -Nome \"Nome do Dispositivo\"\n```\n\n### Onde o relatório é salvo?\n\nAutomaticamente na Desktop: `C:\\Users\\seu-usuario\\Desktop\\Relatorio_Rede.html`\n\n### Posso alterar onde salva?\n\nSim, edite a linha:\n```powershell\n$CaminhoArquivo = \"C:\\outro\\caminho\\Relatorio_Rede.html\"\n```\n\n### Quanto tempo demora para executar?\n\nNormalmente **1-3 segundos** (depende da velocidade da rede).\n\n## ❌ Problemas\n\n### \"O acesso ao caminho foi negado\"\n\n**Causa:** Sem permissão para escrever na Desktop\n\n**Solução:**\n1. Execute como Administrador\n2. Ou altere o caminho para outra pasta com permissão\n\n### \"Test-Connection: Nome de host inválido\"\n\n**Causa:** IP incorreto ou dispositivo não existe\n\n**Solução:**\n1. Verifique o IP correto\n2. Teste com `ping 192.168.1.1` no CMD\n3. Certifique-se que está conectado à rede\n\n### \"O script não funciona\"\n\n**Causa 1:** Política de execução bloqueada\n```powershell\nSet-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser\n```\n\n**Causa 2:** PowerShell versão muito antiga\n```powershell\n$PSVersionTable  # Verifique a versão\n```\n\n**Causa 3:** Caminho com espaços ou caracteres especiais\n- Mova o script para pasta sem caracteres especiais\n\n### HTML abre em branco\n\n**Causa:** Navegador não suporta\n\n**Solução:**\n- Use Edge, Chrome ou Firefox\n- Tente atualizar o navegador\n\n### O script congela/não responde\n\n**Causa:** Dispositivo offline demora para responder\n\n**Solução:**\n1. Aguarde (pode levar alguns segundos)\n2. Ou altere o timeout no código (avançado)\n\n## 🔐 Segurança\n\n### O script acessa minhas senhas?\n\nNão. O script apenas:\n- Testa conectividade via PING\n- Escreve um arquivo HTML\n- Nunca acessa senhas ou dados sensíveis\n\n### É seguro executar?\n\nSim. O código é aberto e auditável. Verifique você mesmo!\n\n### O script envia dados para internet?\n\nNão, exceto para o IP 8.8.8.8 (Google DNS) para testar internet.\n\n### Preciso de antivírus?\n\nNormalmente não, mas:\n- Alguns antivírus bloqueiam scripts\n- Se tiver problema, adicione às exceções\n\n## 💻 Técnico\n\n### Como funciona a função Get-StatusHTML?\n\n```powershell\n# Testa a conexão\n$Status = Test-Connection -ComputerName $IP -Count 1 -Quiet\n\n# Retorna HTML com status\nif ($Status) {\n    return \"<tr>...<span style='color:green;'>ONLINE</span>...\"\n} else {\n    return \"<tr>...<span style='color:red;'>OFFLINE</span>...\"\n}\n```\n\n### Posso usar em scripts automation?\n\nSim! O script retorna resultados que podem ser capturados:\n\n```powershell\n$output = .\\Check-upTI.ps1\n# Processar $output conforme necessário\n```\n\n### Como adicionar logs?\n\nAdicione no final do script:\n\n```powershell\n$HTML_Final | Out-File -FilePath $LogPath -Append\nWrite-Host \"Log salvo em: $LogPath\"\n```\n\n## 📊 Customização\n\n### Como alterar cores?\n\nEdite as linhas com estilos CSS:\n\n```powershell\n# Encontre e altere:\nstyle='color:green;'  # Altere para outra cor\nstyle='color:red;'    # Altere para outra cor\n```\n\nCores disponíveis: `red`, `green`, `blue`, `orange`, `purple`, `#ff0000`, etc.\n\n### Como adicionar mais informações ao relatório?\n\nEdite a geração do HTML:\n\n```powershell\n$MaisInfo = \"<p>Informação adicional</p>\"\n$HTML_Final = $Header + $Titulo + $MaisInfo + $TabelaInicio + ...\n```\n\n### Posso exportar para PDF?\n\nNão nativamente, mas:\n1. Abra o HTML no navegador\n2. Pressione Ctrl+P\n3. Escolha \"Salvar como PDF\"\n\n## 🤝 Contribuição\n\n### Como contribuir?\n\nVeja [CONTRIBUTING.md](CONTRIBUTING.md)\n\n### Como reportar um bug?\n\nAbra uma [Issue](https://github.com/seu-usuario/check-up-ti/issues) com:\n- Descrição clara\n- Passos para reproduzir\n- Sua versão do Windows/PowerShell\n- Screenshots (se aplicável)\n\n### Posso criar minha própria versão?\n\nSim!
